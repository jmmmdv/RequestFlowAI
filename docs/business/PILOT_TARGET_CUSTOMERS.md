# Pilot target customers

Practical list of early customer types for the **RequestFlowAI Early Access Pilot**. Use this to
prioritize outreach and pick a strong first question per industry.

**Related:** [CUSTOMER_OUTREACH_MESSAGES.md](CUSTOMER_OUTREACH_MESSAGES.md) ·
[PILOT_VALIDATION_PLAN.md](PILOT_VALIDATION_PLAN.md) · [PILOT_OFFER.md](PILOT_OFFER.md)

**Legend:** **Good fit** = likely pain + fast decision. **Weak fit** = harder pilot unless you have a warm intro or niche angle.

---

## Local service businesses

| | |
|---|---|
| **Examples** | Cleaning, landscaping, pest control, salons, pet grooming |
| **Why they may need RequestFlowAI** | Many small jobs, repeat clients, requests by phone/text/email |
| **Likely pain** | Owner is in the field; messages pile up; urgent jobs get mixed with quotes |
| **First question** | "When you're on a job and a client texts a new request, how do you make sure it doesn't get lost?" |
| **Pilot fit** | **Good fit** — simple workflow, owner decides quickly |

---

## Transportation / limo companies

| | |
|---|---|
| **Examples** | Car service, airport transfers, party buses, local courier |
| **Why they may need RequestFlowAI** | Bookings change last minute; dispatch needs one queue |
| **Likely pain** | Double bookings, missed texts, unclear pickup details |
| **First question** | "How do booking requests get from the client to the driver today?" |
| **Pilot fit** | **Good fit** if volume is steady — **weak** if they already live in dispatch software |

---

## Medical / dental / clinic offices

| | |
|---|---|
| **Examples** | Dental, chiropractic, med spa, small specialty clinic |
| **Why they may need RequestFlowAI** | Appointment changes, billing questions, referral paperwork |
| **Likely pain** | Front desk overloaded; non-urgent requests buried in phone tag |
| **First question** | "Besides scheduling software, where do patient questions land that aren't a live appointment?" |
| **Pilot fit** | **Weak fit** for MVP — HIPAA/privacy and existing practice management systems add friction. Only pursue with a warm intro and clear non-PHI use (e.g. general inquiries, not clinical records) |

---

## Home repair / service companies

| | |
|---|---|
| **Examples** | HVAC, plumbing, electrical, roofing, handyman |
| **Why they may need RequestFlowAI** | Emergency vs estimate requests; photos and addresses in texts |
| **Likely pain** | After-hours voicemails; no shared view of what's open vs scheduled |
| **First question** | "How do you separate emergency calls from quote requests when you're busy on site?" |
| **Pilot fit** | **Good fit** — high pain, owner-operator friendly |

---

## Small law / accounting offices

| | |
|---|---|
| **Examples** | Solo attorney, small CPA firm, bookkeeper serving multiple clients |
| **Why they may need RequestFlowAI** | Clients send documents and questions async; matter-based tracking |
| **Likely pain** | Email threads per client; deadlines; "did we respond?" anxiety |
| **First question** | "When a client emails a new request, how does your team decide priority and who owns the reply?" |
| **Pilot fit** | **Good fit** for intake + triage — **weak** if they need matter management / conflict checks on day one |

---

## Education / tutoring businesses

| | |
|---|---|
| **Examples** | Tutoring center, music lessons, test prep, small training provider |
| **Why they may need RequestFlowAI** | Parents and students ask scheduling, billing, and progress questions |
| **Likely pain** | Seasonal spikes; requests across email, WhatsApp, and forms |
| **First question** | "During busy season, how do you keep parent messages from slipping through?" |
| **Pilot fit** | **Good fit** — clear volume swings; often one admin + owner |

---

## Property management companies

| | |
|---|---|
| **Examples** | Small PM shop, HOA manager, short-term rental operator |
| **Why they may need RequestFlowAI** | Tenant maintenance, owner approvals, vendor coordination |
| **Likely pain** | Every unit is a thread; urgent leaks vs routine requests |
| **First question** | "When a tenant reports an issue, what's the path from first message to assigned vendor?" |
| **Pilot fit** | **Good fit** at small scale — **weak** if they need full work-order + accounting integration immediately |

---

## Priority order for first 10 conversations

Suggested outreach order for fastest learning:

1. Home repair / service companies  
2. Local service businesses  
3. Transportation / limo (if no heavy dispatch lock-in)  
4. Education / tutoring  
5. Small law / accounting  
6. Property management (small portfolios)  
7. Medical / dental — only with clear non-PHI angle and warm trust  

---

## Disqualify fast when you hear

- "We need Salesforce/ServiceNow integration first"
- "Legal/IT must approve every vendor" with no champion
- "We get 2 requests a month" (too low volume)
- "We only want AI to replace staff" (misaligned — rule-based today)
- No one willing to check a dashboard weekly
