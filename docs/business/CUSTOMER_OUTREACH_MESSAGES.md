# Customer outreach messages

Copy-paste templates for founder-led pilot outreach. Personalize the bracketed parts before sending.

**Related:** [PILOT_OFFER.md](PILOT_OFFER.md) · [CUSTOMER_DISCOVERY_QUESTIONS.md](CUSTOMER_DISCOVERY_QUESTIONS.md) ·
[PILOT_TARGET_CUSTOMERS.md](PILOT_TARGET_CUSTOMERS.md)

**Tone:** friendly, honest, early pilot — not a polished enterprise pitch.

---

## Short SMS / text message

Hi [Name] — I'm building RequestFlowAI, a simple tool that helps businesses collect client requests in one place and see what’s urgent. I’m running a small 30-day early pilot and setup is free for the first few businesses. Would you be open to a quick 15-minute call this week? No pressure — just learning if this solves a real problem for [business type] like yours.

## Email version

**Subject:** Quick question about how you handle client requests

Hi [Name],

I'm [Your name], founder of **RequestFlowAI**. We're building a lean tool that helps small businesses:

- collect client requests through one simple link
- organize and prioritize what came in
- track status in a small admin dashboard

We're **not** a big enterprise platform — this is an early **30-day pilot** while we validate the product with real businesses. Setup is **free for the first pilot customers**, and I'm looking for honest feedback more than a perfect demo.

Would you have **15 minutes** for a short conversation? I'd love to hear how [Business name] handles requests today and whether a tool like this would actually help.

If it's not a fit, no worries — even a quick "here's what we do instead" helps a lot.

Thanks,  
[Your name]  
[Email] · [Phone optional]

## Tips before you send

| Do | Avoid |
|---|---|
| Use their business name and one real detail | Mass-blast identical spam |
| Say "early pilot" and "free setup for first customers" | Promise AI magic or full automation |
| Ask for 15 minutes, not an hour | Attach a long deck on first touch |
| Log every reply in [VALIDATION_TRACKER_TEMPLATE.md](VALIDATION_TRACKER_TEMPLATE.md) | Build custom features before the call |

After a yes: send [CUSTOMER_DISCOVERY_QUESTIONS.md](CUSTOMER_DISCOVERY_QUESTIONS.md) mentally, book the call, and offer [PILOT_OFFER.md](PILOT_OFFER.md) only if fit looks good.
