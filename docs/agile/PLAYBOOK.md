# Agile delivery playbook

## Working agreement

- Sprint length: one week while learning.
- Daily check: what changed, what is next, what is blocked?
- Work-in-progress limit: one item per learner.
- Demo working software; documents alone are not a vertical slice.

## Definition of ready

The story names a user, outcome, acceptance examples, dependencies, and a small enough slice.

## Definition of done

Code is reviewed; unit, integration, and E2E checks pass; docs and operational impact are updated;
security/privacy risks are considered; the feature is deployable and demonstrated.

## Sample sprint

Goal: “A learner can manage work through a resilient REST API.” Select create/read/update/delete,
validation, API tests, and one happy-path UI test. Track cycle time, escaped defects, flaky-test rate,
and whether the sprint goal was achieved—not developer activity.

Retrospective prompts: what helped flow, where did uncertainty wait, what one experiment will we run
next sprint? Turn the chosen experiment into a backlog item with an owner.
