# Pilot customer onboarding

Use this guide when inviting the first five pilot teams to RequestFlow AI.

## Before the call

1. Confirm production Cognito, API, and browser configuration are deployed.
2. Run the drills in [LAUNCH-CHECKLIST.md](LAUNCH-CHECKLIST.md).
3. Prepare a private request-portal link for the pilot organization.

## First session (30 minutes)

1. **Problem fit** — Ask where requests arrive today (email, WhatsApp, forms) and what gets lost.
2. **Sign up** — Founder creates an account through **Start free** and completes workspace setup.
3. **Share the form** — Copy the public request link from **Request portal** and send a test request.
4. **Review the inbox** — Show category, priority, summary, next action, and linked work item.
5. **Track work** — Move one item from backlog to in progress to done.
6. **Team path** — Explain invitations and roles if they will add a teammate during the pilot.

## What to promise honestly

- Rule-based triage, not an AI chatbot.
- Pilot-ready foundation, not a finished enterprise product.
- Production abuse controls exist in code; WAF and monitoring still belong in your cloud setup.

## After the session

- Record feedback in your issue tracker.
- Note blockers for billing, identity, or intake.
- Schedule a two-week check-in on request volume and workflow fit.

## Support handoff

Share [SUPPORT.md](SUPPORT.md) and [PRIVACY.md](PRIVACY.md) before they send customer data through the portal.
